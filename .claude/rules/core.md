# Core Rules

> Эти правила применяются ВСЕГДА. Нарушение = баг.

## 1. GIT — ПУШИ ВСЕГДА

```bash
git add . && git commit -m "тип: описание" && git push
```

**После каждого изменения.** Заказчики на разных устройствах — без push потеряют работу.

Типы коммитов:
- `feat:` — новая функция
- `fix:` — исправление бага
- `refactor:` — рефакторинг
- `style:` — стили, форматирование
- `docs:` — документация

## 2. ПРОВЕРЯЙ ПЕРЕД КОММИТОМ

```bash
npm run build
```

Ошибки? Исправь сам. Не коммить сломанное.

## 3. ДЕЛАЙ ВСЁ САМ

Заказчики НЕ программисты. Они не знают:
- git, терминал, npm
- TypeScript, React
- SQL, Supabase

**Не проси их:**
- запускать команды
- проверять код
- смотреть терминал

**Говори просто:**
- "Готово! Добавил кнопку."
- "Исправил. Посты теперь сохраняются."

## 4. НЕ ЛОМАЙ

- Прочитай файл перед изменением
- Меняй только то, что просят
- Не рефактори "заодно"
- Не добавляй "на будущее"

## 5. ОТКАТ

Сломал? Откати:

```bash
# Незакоммиченные изменения
git checkout -- .

# Последний коммит
git revert HEAD && git push
```

Скажи заказчику что произошло.

## 6. КОНФЛИКТЫ

```bash
git stash
git pull
git stash pop
```

Конфликт? Реши сам, не проси заказчика.

## 7. НЕ ИСПОЛЬЗУЙ

- Next.js (только Vite)
- Express/Node backend (только Edge Functions)
- CSS файлы (только Tailwind)
- Inline styles (только className)
- any в TypeScript
- Классовые компоненты

## 8. БЕЗОПАСНОСТЬ

- `.env.local` никогда в git
- Service Role Key только на сервере
- RLS на каждой таблице
- Валидация пользовательского ввода
